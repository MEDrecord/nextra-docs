## Vercel (Frontend Hosting)

All MEDrecord frontend applications are deployed on [Vercel](https://vercel.com).

### Why Vercel?

- **Edge Network** - Global CDN with automatic caching
- **Zero-config deployments** - Git push triggers automatic builds
- **Preview deployments** - Every PR gets a unique preview URL
- **Serverless functions** - API routes scale automatically
- **Analytics** - Built-in performance monitoring

### Deployment Flow

1. Push code to GitHub branch
2. Vercel automatically builds and deploys
3. Preview URL generated for review
4. Merge to main triggers production deployment

### Environment Variables

Vercel manages environment variables per environment:
- `GATEWAY_URL` - Backend API endpoint
- `NEXT_PUBLIC_*` - Client-side variables
- Secrets are encrypted and never exposed to client

## Java Backend (Gateway & Services)

The backend is built with Spring Boot and provides the core business logic and data layer.

### Technology Stack

| Component | Technology |
|-----------|------------|
| Framework | Spring Boot 3.x |
| Language | Java 21 |
| Database | PostgreSQL |
| Clinical Data | OpenEHR / FHIR |
| Auth | OAuth 2.0 / HMAC |
| Build | Maven |
| Deployment | Kubernetes |

### Gateway Architecture

The Gateway is the single entry point for all frontend applications:

```java
// All requests go through the Gateway
Frontend → Gateway → Internal Services → Database
```

**Key responsibilities:**
- Authentication and authorization
- Request routing to internal services
- Rate limiting and throttling
- Audit logging
- HMAC signature verification

### API Authentication

All frontend-to-backend communication uses HMAC signatures:

```typescript
// Frontend: Sign request
const signature = createHmacSignature(payload, timestamp, secretKey)

// Backend: Verify signature
@PreAuthorize("hasValidHmacSignature()")
public ResponseEntity<?> handleRequest() { ... }
```

See [Gateway Documentation](/docs/products/gateway) for full API reference.

### Development Workflow

1. **Local development**: Run Gateway locally with Docker Compose
2. **Feature branches**: Deploy to test environment
3. **Code review**: PR with automated tests
4. **Staging**: Integration testing
5. **Production**: Blue-green deployment

### Connecting Frontend to Backend

Frontend applications **never** connect directly to databases or internal services.

```typescript
// CORRECT: Call Gateway API
const response = await fetch(`${GATEWAY_URL}/api/patients`, {
  headers: {
    'X-Gateway-Signature': signature,
    'X-Timestamp': timestamp
  }
})

// WRONG: Direct database access
// import { db } from '@/lib/database' // Never do this!
```
