# Compliance for Developers

This guide helps developers build compliant features for the MEDrecord platform. For full compliance documentation, see the [ISMS section](/isms).

## Before You Code Checklist

Before starting development on any feature, verify:

- [ ] Feature requirements are documented and approved
- [ ] Data classification is defined (PHI, PII, or general)
- [ ] Security requirements are identified
- [ ] Testing approach is planned

## Security Checklist

Every code change must follow these security principles:

### Data Handling

- [ ] No PHI/PII in console logs or error messages
- [ ] All user input validated with zod schemas
- [ ] SQL queries parameterized (no string concatenation)
- [ ] Sensitive data encrypted at rest and in transit

### Authentication & Authorization

- [ ] All API routes verify `auth.id` from Gateway
- [ ] Role-based access control enforced
- [ ] Session tokens handled securely (HttpOnly cookies)
- [ ] No credentials in client-side code

### API Security

- [ ] Rate limiting configured for endpoints
- [ ] CORS headers properly configured
- [ ] Input size limits enforced
- [ ] Error responses don't leak internal details

## Code Patterns

### Gateway Authentication (Required)

```typescript
// Server-side only - verify every request
import { getUser } from '@/lib/auth/server'

export async function GET(request: Request) {
  const user = await getUser()
  if (!user) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 })
  }
  // Proceed with authenticated user
}
```

### Input Validation (Required)

```typescript
import { z } from 'zod'

const schema = z.object({
  patientId: z.string().uuid(),
  message: z.string().min(1).max(1000),
})

const result = schema.safeParse(input)
if (!result.success) {
  return Response.json({ error: 'Invalid input' }, { status: 400 })
}
```

### Audit Logging

```typescript
// Log significant actions for compliance
await auditLog({
  action: 'patient.view',
  userId: user.id,
  resourceId: patientId,
  timestamp: new Date(),
})
```

## Platform Certifications

The MEDrecord platform maintains the following certifications:

| Standard | Focus | Developer Impact |
|----------|-------|------------------|
| NEN 7510 | Healthcare security | Access control, encryption, logging |
| ISO 13485 | Quality management | Documentation, testing, traceability |
| ISO 62304 | Software lifecycle | Design controls, verification |
| GDPR | Data protection | Consent, data minimization, retention |

For detailed certification requirements, see [ISMS Policies](/isms/policies).

## Pre-PR Checklist

Before creating a pull request:

- [ ] Code follows [architecture patterns](/developer/architecture/overview)
- [ ] All backend operations are server-side only
- [ ] Input validation implemented
- [ ] Error handling doesn't expose sensitive info
- [ ] No hardcoded credentials or secrets
- [ ] Unit tests cover security-critical paths
- [ ] Accessibility requirements met (WCAG 2.1 AA)
