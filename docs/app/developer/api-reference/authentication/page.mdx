# Authentication

The MEDrecord Gateway API uses HMAC signature-based authentication for secure access across all platform products.

> For a complete overview of the Gateway architecture, see [Gateway Documentation](/docs/products/gateway).

## Quick Reference

### Required Headers

| Header | Description |
|--------|-------------|
| `X-Gateway-Signature` | HMAC-SHA256 signature of the request |
| `X-Timestamp` | Unix timestamp of the request |
| `X-Tenant-Id` | Your organization's tenant ID |

### Signature Generation

```javascript
const crypto = require('crypto');

function generateSignature(method, path, timestamp, body, secret) {
  const payload = `${method}${path}${timestamp}${JSON.stringify(body || {})}`;
  return crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
}
```

## Example Request

```bash
curl -X POST https://auth.healthtalk.ai/v1/messages \
  -H "Content-Type: application/json" \
  -H "X-Gateway-Signature: abc123..." \
  -H "X-Timestamp: 1678901234" \
  -H "X-Tenant-Id: your-tenant-id" \
  -d '{"patientId": "123", "message": "Hello"}'
```

## API Credentials

API credentials are managed in the admin dashboard of your product:

1. Navigate to **Settings > API**
2. Click **Generate New Key**
3. Store the secret securely (it won't be shown again)
4. Set IP restrictions if needed

## Token Validity

- Signatures are valid for **5 minutes** from the timestamp
- Clock skew tolerance is **30 seconds**
- Requests with expired timestamps are rejected

## Security Checklist

For developers integrating with the Gateway API:

- [ ] Never expose API secrets in client-side code
- [ ] Rotate API keys regularly (quarterly recommended)
- [ ] Use IP whitelisting for production systems
- [ ] Monitor API usage for anomalies
- [ ] Implement proper error handling for auth failures
