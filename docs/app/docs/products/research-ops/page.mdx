# ResearchOps

ResearchOps is the operating system for research projects. It helps research teams secure funding faster and manage complex collaborative projects with confidence — from funding discovery and consortium building to proposal development, project coordination, and reporting.

## Grant Engineering Philosophy

ResearchOps adopts a **Cognitive Offloading** approach. This is not "AI chatting" but "AI grant engineering":

### Core Principles

| Principle | Description |
|-----------|-------------|
| **Human-in-the-loop** | AI executes, human verifies and approves before proceeding |
| **Phase-gated workflow** | 4 phases with mandatory quality gates between each |
| **Locked artifacts** | Critical decisions lock after approval - everything downstream builds on them |
| **Structured outputs** | Every step produces specific .md or .xlsx knowledge artifacts |
| **Consistency tracking** | SCRATCHPAD.md serves as "shared brain" for cross-checks |

### The Key Distinction

```
Strategy + Judgment = HUMAN
Everything else = .md files
```

AI extends capacity by handling analysis, calculation, drafting, and verification. Humans retain judgment for:

- Domain knowledge application
- Personal experience integration
- Network leverage (partners, experts)
- Business and career alignment
- Final decision making and argumentation

## Phase-Gated Workflow

ResearchOps uses a 4-phase model with mandatory quality gates:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ResearchOps Proposal Engine                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Phase 1              Phase 2              Phase 3        Phase 4   │
│  KNOWLEDGE BASE  →   WRITING INFRA    →   WRITING    →   ASSEMBLY  │
│  (8 decisions)       (structures)         (sections)     (submit)   │
│                                                                     │
│  ┌─────┐            ┌─────┐              ┌─────┐        ┌─────┐    │
│  │GATE │            │GATE │              │GATE │        │GATE │    │
│  │P11  │            │     │              │     │        │FINAL│    │
│  └─────┘            └─────┘              └─────┘        └─────┘    │
│                                                                     │
│  Human approves each step before AI proceeds to next                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Phase 1: Knowledge Base

Build the foundation through 8 structured decisions:

| Step | Command | Output | Description |
|------|---------|--------|-------------|
| P1 | `run P1` | `01-call-context.md` | Contextual EU call study |
| P2 | `run P2` | `02-compliance-checklist.md` | Admissibility and eligibility |
| P3 | `run P3` | `03-evaluation-criteria.md` | Extract scoring criteria and weights |
| P4 | `run P4` | `04-technical-requirements.md` | Page limits, character limits, formats |
| P5 | `run P5` | `05-partnership-requirements.md` | Partner info needs, PIF template |
| P6 | `run P6` | `06-logic-map.md` | **LOCKS after approval** |
| P7 | `run P7` | `07-timeline-milestones.md` | Built on locked P6 |
| P8 | `run P8` | `08-writing-strategy-matrix.xlsx` | Bridge to Phase 2 |
| P9 | `run P9` | `09-qa-findings.md` | Quality checkpoint |
| P10 | `run P10` | `10-patterns.md` | Funded proposal patterns (optional) |
| P11 | `run P11` | `11-coherence-gate.md` | **MANDATORY PASS** |

### Phase 2: Writing Infrastructure

| Step | Output | Description |
|------|--------|-------------|
| P12 | `12-evidence-library.md` | Supporting evidence and citations |

### Phase 3: Writing

Section drafts based on locked infrastructure from Phase 1-2.

### Phase 4: Assembly and Submit

Final assembly, formatting, compliance check, and submission.

## Lock Points and Quality Gates

### Lock Points

Critical outputs lock after human approval. Everything downstream builds on locked artifacts.

| Artifact | When It Locks | What Depends On It |
|----------|---------------|-------------------|
| `06-logic-map.md` | After P6 approval | P7-P11, all Phase 2-4 work |
| `11-coherence-gate.md` | After passing | Phase 2 can begin |

### Coherence Gate (P11)

The coherence gate cross-checks ALL P1-P10 outputs for contradictions:

| Check | Description |
|-------|-------------|
| **Eligibility vs Scope** | Does proposal match call requirements? |
| **Timeline vs Budget** | Are milestones properly resourced? |
| **Partners vs Work Packages** | Do capabilities match assignments? |
| **Evaluation vs Content** | Does logic map address all criteria? |

**Verdict**: PASS / CONDITIONAL / FAIL

Must **PASS** before proceeding to Phase 2.

```
User → Gateway → ResearchOps
          │
```
