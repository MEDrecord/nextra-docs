# ConsortiaAI

ConsortiaAI is an AI-powered research grant and consortium management platform for coordinators of European and national R&D projects. It supports the full lifecycle from funding discovery and partner matching to proposal development, consortium agreements, milestone tracking, deliverables, compliance, and reporting.

## Part of MEDrecord

ConsortiaAI is a product within the MEDrecord ecosystem, sharing the same infrastructure, authentication, and design language as HealthTalk, Helpdesk, and other MEDrecord products.

## Multi-Tenancy

ConsortiaAI is multi-tenant, supporting multiple organizations with isolated data and configurations.

### Tenant Structure

```
┌─────────────────────────────────────────────────────────────┐
│                     ConsortiaAI Platform                    │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  Tenant A   │  │  Tenant B   │  │  Tenant C   │         │
│  │  (UMC X)    │  │  (Institute)│  │  (Company)  │         │
│  ├─────────────┤  ├─────────────┤  ├─────────────┤         │
│  │ Projects    │  │ Projects    │  │ Projects    │         │
│  │ Partners    │  │ Partners    │  │ Partners    │         │
│  │ Users       │  │ Users       │  │ Users       │         │
│  │ Settings    │  │ Settings    │  │ Settings    │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### Tenant Isolation

| Aspect | Isolation Level |
|--------|-----------------|
| **Data** | Fully isolated - no cross-tenant data access |
| **Users** | Per-tenant user management with roles |
| **Projects** | Tenant-scoped, shareable via consortium invites |
| **Settings** | Per-tenant configuration |
| **Integrations** | Tenant-specific API keys (Cal.com, Stripe, Document Services) |

### Gateway Integration

ConsortiaAI uses Gateway for multi-tenant routing:

```
User → Gateway → ConsortiaAI
          │
          ├── Validates auth.sid cookie
          ├── Extracts tenant context (X-Tenant-Id)
          ├── Validates tenant membership
          └── Returns user with tenant role
```

**Headers from Gateway:**

| Header | Description |
|--------|-------------|
| `X-User-Id` | Authenticated user identifier |
| `X-Tenant-Id` | Active tenant/organization |
| `X-Tenant-Role` | User role within tenant |

### Tenant Roles

| Role | Description | Permissions |
|------|-------------|-------------|
| **Owner** | Tenant administrator | Full access, manage users, billing |
| **Coordinator** | Project coordinator | Create projects, manage partners, full project access |
| **Partner** | External consortium partner | Invited project access only |
| **Viewer** | Read-only access | View projects, no edit rights |

### Cross-Tenant Collaboration

Research consortia often span multiple organizations. ConsortiaAI handles this via:

1. **Consortium Invites** - Invite partners from other tenants to specific projects
2. **Shared Workspaces** - Project-level access for cross-tenant collaboration
3. **Federated Identity** - Partners authenticate via their own tenant

```
┌──────────────────────────────────────────────────────────────┐
│                    Project: Health-AI-2025                   │
├──────────────────────────────────────────────────────────────┤
│  Coordinator (Tenant A)  ←→  Partner (Tenant B)              │
│  Coordinator (Tenant A)  ←→  Partner (Tenant C)              │
│  Coordinator (Tenant A)  ←→  External Partner (Email invite) │
└──────────────────────────────────────────────────────────────┘
```

## Overview

ConsortiaAI helps research coordinators manage complex multi-partner projects efficiently. The platform uses AI agents to automate repetitive tasks while keeping humans in control of strategic decisions.

### Target Users

- Research coordinators at University Medical Centers
- Research Support Office staff
- Project managers for Horizon Europe, Eurostars, ZonMw, NWO projects
- Innovation managers at healthcare organizations

### Key Capabilities

- **Funding Intelligence** - AI-powered call monitoring and eligibility matching
- **Consortium Builder** - Partner discovery, outreach automation, LOI tracking
- **Proposal Workspace** - Collaborative editing with compliance checks
- **Contracts and Agreements** - Templates, negotiation, custom e-signatures
- **Planning and Scheduling** - Cal.com integration for partner meetings and deadlines
- **Project Execution** - Work packages, milestones, deliverables tracking
- **Financials and Payments** - Budget tracking, success fees, invoicing via Stripe
- **Communication Tracking** - WhatsApp, email integration for audit trail
- **Project Documentation** - Per-proposal docs (Nextra-style) with version control
- **Reporting Suite** - Periodic reports, financial reporting, KPI dashboards

## Custom E-Signature (ConsortiaSign)

ConsortiaAI includes a custom-built e-signature system, replacing third-party dependencies like PandaDoc.

### Architecture

```
┌────────────────────────────────────────────────────────────────┐
│                      ConsortiaSign                             │
├────────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │   Document   │  │   Signing    │  │    Audit     │         │
│  │   Prepare    │  │   Workflow   │  │    Trail     │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
│         │                 │                  │                 │
│         ▼                 ▼                  ▼                 │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │  PDF Render  │  │  Email/SMS   │  │  Timestamp   │         │
│  │  + Fields    │  │  Invites     │  │  Authority   │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
└────────────────────────────────────────────────────────────────┘
```

### Features

| Feature | Implementation |
|---------|----------------|
| **Document Prep** | PDF.js for rendering, drag-drop signature field placement |
| **Signing Flow** | Email invites with secure magic links for external signers |
| **Canvas Signature** | Draw or type signature, stored securely |
| **Authentication** | Gateway auth for internal, magic link for external signers |
| **Audit Trail** | Timestamp, IP address, user agent, document hash |
| **Certificate** | Signed PDF with embedded digital certificate |
| **Storage** | Signed documents in Supabase Storage, encrypted |

### Signing Workflow

```
Coordinator → Prepare Document → Add Signature Fields
                                        │
                                        ▼
                              Send to Signers (Email)
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
              Signer A            Signer B            Signer C
              (Internal)          (Partner)           (External)
              Gateway Auth        Gateway Auth        Magic Link
                    │                   │                   │
                    └───────────────────┼───────────────────┘
                                        │
                                        ▼
                              All Signed → Generate Certificate
                                        │
                                        ▼
                              Store in Document Vault
```

### Legal Compliance

| Requirement | Solution |
|-------------|----------|
| **eIDAS (EU)** | Advanced Electronic Signature (AdES) compliant |
| **Timestamping** | RFC 3161 timestamp authority integration |
| **Document Integrity** | SHA-256 hash before and after signing |
| **Identity Verification** | Gateway auth + optional ID verification |
| **Audit Log** | Immutable log of all signing events |

### Qualified Signatures (QES)

For grants requiring Qualified Electronic Signatures, ConsortiaSign integrates with Trust Service Providers:

| Provider | Use Case |
|----------|----------|
| **Signicat** | EU-wide QES for formal grant agreements |
| **GlobalSign** | International digital certificates |
