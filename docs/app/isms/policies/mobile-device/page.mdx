## Purpose

This policy establishes security requirements for mobile devices used to access MEDrecord information and systems, including company-owned devices and personal devices (BYOD).

## Scope

This policy applies to all smartphones, tablets, laptops, and other portable devices that:
- Are owned by MEDrecord
- Access MEDrecord email, data, or systems
- Store MEDrecord information

## Policy Statement

### Device Requirements

All mobile devices accessing company resources must meet these minimum requirements:

| Requirement | Standard |
|-------------|----------|
| Operating System | Current or previous major version (iOS 16+, Android 13+) |
| Screen Lock | PIN (6+ digits), password, or biometric |
| Auto-Lock | Maximum 5 minutes of inactivity |
| Encryption | Full device encryption enabled |
| Security Updates | Applied within 14 days of release |
| Antivirus | Required on Android devices |

### Company-Owned Devices

Company devices are:
- Pre-configured with required security settings
- Enrolled in Mobile Device Management (MDM)
- Subject to remote wipe capability
- For business use primarily
- Must be returned upon termination

### Personal Devices (BYOD)

Employees may use personal devices for work if they:
- Meet minimum security requirements above
- Enroll in MDM for company email/apps access
- Accept that company data may be remotely wiped
- Keep work and personal data separated
- Allow IT to verify security compliance

**BYOD Enrollment:**
1. Contact IT to register your device
2. Install required MDM profile
3. Configure work profile/container
4. Complete security verification

### Prohibited Activities

Do not:
- Jailbreak or root devices
- Disable security features
- Install apps from untrusted sources
- Store patient/healthcare data locally
- Use public charging stations (juice jacking risk)
- Connect to untrusted networks without VPN
- Share device access credentials

### Application Security

**Approved Apps:**
- Microsoft Outlook (email)
- Microsoft Teams (communication)
- Microsoft Authenticator (MFA)
- Company-approved MDM apps

**App Installation:**
- Only install apps from official stores (App Store, Google Play)
- Review app permissions before installation
- Remove unused apps regularly
- Keep all apps updated

### Lost or Stolen Devices

If your device is lost or stolen:

1. **Immediately** report to IT Support / Security Officer
2. Attempt to locate using Find My Device
3. IT will initiate remote wipe if necessary
4. Change passwords for accounts accessed on device
5. File police report if stolen

**Contact:** IT Support via [Helpdesk](https://helpdesk.healthtalk.ai) or emergency: [security@medrecord.nl](mailto:security@medrecord.nl)

### Travel Guidelines

When traveling:
- Enable device tracking (Find My iPhone/Android)
- Use VPN on all networks
- Be aware of shoulder surfing
- Keep devices in carry-on luggage (never checked bags)
- Use hotel safes when available
- Be cautious at border crossings (devices may be inspected)

### Data Handling

| Data Type | Mobile Access |
|-----------|---------------|
| Public | Allowed |
| Internal | Allowed with encryption |
| Confidential | View only, no local storage |
| Healthcare/Patient | Approved apps only, no screenshots |

## Responsibilities

| Role | Responsibility |
|------|----------------|
| Employees | Follow policy, report lost/stolen devices |
| IT Team | Configure MDM, support device enrollment |
| Security Officer | Define requirements, audit compliance |
| Management | Ensure team compliance |

## Compliance

- MDM compliance is monitored automatically
- Non-compliant devices may lose access to company resources
- Regular security scans are performed
- Annual policy acknowledgment required
