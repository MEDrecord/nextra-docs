## Purpose

This policy defines the security requirements for employees working remotely to ensure the protection of MEDrecord information and systems when accessed from outside the office.

## Scope

This policy applies to all employees, contractors, and third parties who work remotely or access company systems from locations outside MEDrecord offices.

## Policy Statement

### General Requirements

Remote workers must:
- Use only company-approved devices or properly secured personal devices
- Connect through approved VPN when accessing internal resources
- Ensure physical security of devices and documents
- Maintain confidentiality of sensitive information
- Follow all other MEDrecord security policies

### Network Security

| Requirement | Standard |
|-------------|----------|
| VPN | Required for all internal resource access |
| Wi-Fi | Use WPA3/WPA2 encryption; avoid public Wi-Fi for sensitive work |
| Home Network | Change default router passwords; keep firmware updated |
| Firewall | Enable device firewall at all times |

### Physical Security

**Workspace Requirements:**
- Work in a private area where screens cannot be viewed by others
- Lock screen when stepping away (Windows: Win+L, Mac: Ctrl+Cmd+Q)
- Do not leave devices unattended in public places
- Store physical documents securely; shred when no longer needed
- Use privacy screens in public spaces

**Device Security:**
- Enable full disk encryption
- Use strong passwords/biometrics for device access
- Keep devices physically secure when traveling
- Never leave devices in vehicles

### Data Handling

**Permitted:**
- Access company data through approved applications
- Use company cloud storage (SharePoint, OneDrive)
- Video calls on approved platforms (Teams, Zoom)

**Not Permitted:**
- Store sensitive data on personal devices
- Use personal cloud storage for company data
- Share screens showing patient/healthcare data in public
- Print sensitive documents at home without secure disposal

### Communication

- Use company email for business communications
- Use approved messaging platforms (Microsoft Teams)
- Verify identity before sharing sensitive information
- Be cautious of phishing attempts (increased risk when remote)

### Equipment

Company-provided equipment:
- Must be used for work purposes only
- Must be returned upon termination
- Must be kept updated with latest patches
- Must have approved security software installed

Personal devices (BYOD):
- Must meet minimum security requirements
- Must be enrolled in mobile device management if accessing email
- Must have separate work profile where possible
- See [Mobile Device Policy](/isms/policies/mobile-device)

## Incident Reporting

Report immediately if:
- Device is lost or stolen
- Suspicious activity is observed
- You suspect a security breach
- You accidentally expose sensitive data

Contact: Security Officer or IT Support via [Helpdesk](https://helpdesk.healthtalk.ai)

## Responsibilities

| Role | Responsibility |
|------|----------------|
| Remote Workers | Follow all policy requirements, report incidents |
| Managers | Ensure team awareness, monitor compliance |
| IT Team | Provide secure remote access tools, support |
| Security Officer | Review policy, audit compliance |

## Compliance

- Remote access logs are monitored
- Periodic security assessments may be conducted
- Non-compliance may result in revoked remote access privileges
- Serious violations may lead to disciplinary action
