## Purpose

This internal privacy policy establishes how MEDrecord handles personal data in compliance with the General Data Protection Regulation (GDPR/AVG) and other applicable privacy regulations.

## Scope

This policy applies to all processing of personal data by MEDrecord employees, including:
- Customer and patient data
- Employee data
- Supplier and partner data
- Website visitor data

## Legal Basis

MEDrecord processes personal data only when there is a valid legal basis:

| Legal Basis | Description | Examples |
|-------------|-------------|----------|
| Consent | Individual has given explicit consent | Marketing emails, cookies |
| Contract | Necessary for contract performance | Service delivery, employment |
| Legal Obligation | Required by law | Tax records, healthcare regulations |
| Vital Interests | Protect life of data subject | Medical emergencies |
| Legitimate Interest | Business need balanced against rights | Security monitoring, fraud prevention |

## Data Protection Principles

All personal data processing must adhere to these principles:

### 1. Lawfulness, Fairness, Transparency
- Process data only with valid legal basis
- Be transparent about data use
- Provide clear privacy notices

### 2. Purpose Limitation
- Collect data only for specified purposes
- Do not use data for incompatible purposes
- Document all processing purposes

### 3. Data Minimization
- Collect only necessary data
- Avoid "nice to have" data collection
- Regularly review data needs

### 4. Accuracy
- Keep data accurate and up to date
- Provide mechanisms for correction
- Verify data at regular intervals

### 5. Storage Limitation
- Retain data only as long as necessary
- Follow retention schedules
- Securely delete when no longer needed

### 6. Integrity and Confidentiality
- Implement appropriate security measures
- Protect against unauthorized access
- Ensure data integrity

### 7. Accountability
- Document all processing activities
- Demonstrate compliance
- Maintain records of processing

## Roles and Responsibilities

| Role | Responsibility |
|------|----------------|
| Data Protection Officer (DPO) | Oversee compliance, advise on DPIA, contact point for authorities |
| Data Controllers | Determine purposes and means of processing |
| Data Processors | Process data on behalf of controller |
| All Employees | Handle personal data responsibly, report breaches |

**DPO Contact:** [dpo@medrecord.nl](mailto:dpo@medrecord.nl)

## Data Subject Rights

Individuals have the following rights regarding their personal data:

| Right | Description | Response Time |
|-------|-------------|---------------|
| Access | Obtain copy of their data | 30 days |
| Rectification | Correct inaccurate data | 30 days |
| Erasure | Request deletion ("right to be forgotten") | 30 days |
| Restriction | Limit processing | 30 days |
| Portability | Receive data in portable format | 30 days |
| Object | Object to certain processing | 30 days |
| Automated decisions | Not be subject to solely automated decisions | 30 days |

### Handling Data Subject Requests

1. Verify identity of requester
2. Log request in tracking system
3. Forward to DPO if complex
4. Respond within 30 days
5. Document response and outcome

## Data Processing Activities

### Healthcare Data (Special Category)

Healthcare data requires additional safeguards:
- Explicit consent or healthcare exemption required
- Encryption mandatory
- Access strictly limited
- Processing logged
- DPIA required for new processing

### Employee Data

Employee personal data is processed for:
- Employment contract administration
- Payroll and benefits
- Performance management
- Legal compliance

### Third-Party Processing

When using third-party processors:
- Data Processing Agreement (DPA) required
- Verify adequate security measures
- Document in processing register
- Regular compliance reviews

## Data Transfers

### Within EU/EEA
- No additional requirements

### Outside EU/EEA
- Only with adequate safeguards:
  - Adequacy decision
  - Standard Contractual Clauses (SCCs)
  - Binding Corporate Rules
- Document legal basis for transfer
- Assess third country laws

## Data Breach Response

In case of a personal data breach:

1. **Detect and Contain** - Stop the breach, preserve evidence
2. **Assess Risk** - Determine scope and severity
3. **Notify DPO** - Within 24 hours of discovery
4. **Authority Notification** - Within 72 hours if high risk (Autoriteit Persoonsgegevens)
5. **Data Subject Notification** - If high risk to individuals
6. **Document** - Record breach details and response

See [Incident Response Procedure](/isms/procedures/incident-response) for detailed steps.

## Privacy by Design

All new projects and systems must incorporate:
- Privacy impact assessment (for high-risk processing)
- Data minimization from the start
- Built-in security controls
- Default privacy settings
- Clear retention and deletion mechanisms

## Training

All employees must:
- Complete privacy awareness training annually
- Understand their responsibilities
- Know how to report incidents
- Stay informed of policy updates

## Compliance Monitoring

- Annual privacy audits
- Regular processing register reviews
- DPIA for high-risk processing
- Documentation of compliance activities
