## Purpose

This policy establishes requirements for managing information security risks in supplier relationships to ensure third parties protect MEDrecord information appropriately.

## Scope

This policy applies to all suppliers and third parties who:
- Access MEDrecord information or systems
- Process data on behalf of MEDrecord
- Provide IT services or infrastructure
- Have access to facilities or networks

## Policy Statement

### Supplier Classification

| Category | Description | Examples |
|----------|-------------|----------|
| Critical | Essential to business operations, access to sensitive data | Cloud providers, EHR vendors |
| Important | Significant impact, limited data access | SaaS tools, consultants |
| Standard | Limited impact, no sensitive data access | Office supplies, general services |

### Pre-Engagement Requirements

Before engaging any supplier:

#### Due Diligence

| Supplier Type | Required Assessments |
|---------------|---------------------|
| Critical | Security questionnaire, audit reports, pentest results, compliance certificates |
| Important | Security questionnaire, compliance certificates |
| Standard | Basic security questionnaire |

#### Required Documentation

- [ ] NDA (Non-Disclosure Agreement)
- [ ] Data Processing Agreement (if processing personal data)
- [ ] Security requirements acknowledgment
- [ ] Insurance certificates (if applicable)
- [ ] Compliance certifications (ISO 27001, SOC 2, NEN 7510)

### Contractual Requirements

All supplier contracts must include:

#### Security Clauses

- Information security requirements
- Data protection obligations
- Incident notification requirements (24-48 hours)
- Right to audit
- Subcontractor approval requirements
- Termination and data return/destruction clauses

#### Healthcare Suppliers (NEN 7510)

Additional requirements for healthcare data:
- NEN 7510 compliance or equivalent
- Data Processing Agreement per GDPR Art. 28
- Specified retention and deletion requirements
- Healthcare-specific incident response
- Audit rights for healthcare data processing

### Supplier Assessment

#### Initial Assessment

| Area | Assessment Items |
|------|------------------|
| Security Governance | Policies, organization, management commitment |
| Access Control | Authentication, authorization, user management |
| Data Protection | Encryption, data handling, privacy |
| Operations | Change management, monitoring, incident response |
| Compliance | Certifications, audit reports, legal compliance |

#### Risk Rating

| Rating | Criteria | Review Frequency |
|--------|----------|------------------|
| High Risk | Critical supplier, sensitive data, gaps identified | Quarterly |
| Medium Risk | Important supplier, moderate data access | Semi-annually |
| Low Risk | Standard supplier, minimal data access | Annually |

### Ongoing Management

#### Regular Reviews

| Review Type | Frequency | Responsible |
|-------------|-----------|-------------|
| Performance review | Quarterly | Contract owner |
| Security assessment | Annual | Security team |
| Compliance verification | Annual | Compliance team |
| Contract review | Annual | Legal/Procurement |

#### Monitoring

- Service level monitoring
- Security incident tracking
- Compliance status updates
- Financial stability checks (critical suppliers)

### Supplier Access Management

#### Access Provisioning

1. Business justification required
2. Minimum necessary access
3. Named individuals only (no shared accounts)
4. Time-limited access
5. Documented approval

#### Access Controls

| Requirement | Standard |
|-------------|----------|
| Authentication | MFA required for all supplier access |
| Authorization | Role-based, least privilege |
| Logging | All access logged and monitored |
| Network | VPN or dedicated secure connection |
| Sessions | Time-limited, activity monitored |

#### Access Termination

- Access removed within 24 hours of contract end
- All credentials revoked
- Data returned or destroyed
- Confirmation obtained

### Incident Management

Suppliers must:
- Report security incidents within 24-48 hours
- Cooperate with investigations
- Provide root cause analysis
- Implement corrective actions

See [Incident Response Procedure](/isms/procedures/incident-response).

### Subcontractor Management

Suppliers must:
- Obtain approval before using subcontractors
- Ensure subcontractors meet same security requirements
- Remain responsible for subcontractor compliance
- Maintain list of approved subcontractors

### Termination and Exit

#### Exit Planning

- Data extraction/return procedures
- Knowledge transfer requirements
- Transition timeline
- Ongoing support during transition

#### Data Handling

- Return of all MEDrecord data
- Certified destruction of data copies
- Confirmation of deletion
- Removal of all access rights

### Supplier Register

All suppliers must be recorded in the [Supplier Register](/isms/registers/suppliers) with:
- Company details
- Contract information
- Risk rating
- Review dates
- Security contacts

## Responsibilities

| Role | Responsibility |
|------|----------------|
| Contract Owner | Day-to-day supplier management |
| Procurement | Contract negotiation, due diligence |
| Security Team | Security assessments, monitoring |
| Legal | Contract review, DPA compliance |
| DPO | Personal data processing oversight |

## Compliance

- Annual supplier audit program
- Compliance verification documented
- Non-compliance escalated to management
- ISO 27001 A.15, NEN 7510 requirements met
