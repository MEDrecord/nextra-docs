## Audit Checklist

Use this checklist during internal or external audits:

### 1. Governance & Leadership

- [ ] Information Security Policy signed by management
- [ ] ISMS scope clearly defined
- [ ] Roles and responsibilities documented
- [ ] Management review minutes available

### 2. Risk Management

- [ ] Risk assessment methodology documented
- [ ] Risk register complete and current
- [ ] Risk treatment plans implemented
- [ ] Risk owners assigned

### 3. Access Control

- [ ] Access control policy documented
- [ ] User access reviews completed (quarterly)
- [ ] Privileged access logs reviewed
- [ ] Terminated user access removed

### 4. Operations Security

- [ ] Change management process followed
- [ ] Backup procedures tested
- [ ] Vulnerability scans completed
- [ ] Security patches applied

### 5. Incident Management

- [ ] Incident response procedure documented
- [ ] Incidents logged and categorized
- [ ] Root cause analysis completed
- [ ] Lessons learned documented

### 6. Business Continuity

- [ ] BCP documented and tested
- [ ] Recovery time objectives met
- [ ] Backup restoration tested
- [ ] DR plan reviewed

### 7. Compliance

- [ ] Legal requirements identified
- [ ] Privacy impact assessments completed
- [ ] Third-party audits completed
- [ ] Corrective actions tracked

## Document Version Control

All ISMS documents are version controlled via Git. Each document displays:

- **Version number** - Semantic versioning (e.g., 2.0)
- **Status** - Draft, In Review, Approved
- **Approved by** - Person who approved the document
- **Approved date** - Date of approval
- **View history** - Link to Git commit history
