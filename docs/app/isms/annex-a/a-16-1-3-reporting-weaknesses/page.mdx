# A.16.1.3 Reporting information security weaknesses

## Control Details

| Field | Value |
|-------|-------|
| **Requirement** | Employees and contractors using the organization's information systems and services shall be required to note and report any observed or suspected information security weaknesses in systems or services. |
| **Applicable** | Yes |
| **Status** | Implemented |

## Implementation

- All personnel trained to recognize and report security weaknesses
- Anonymous reporting option available
- No-blame culture for reporting vulnerabilities
- Responsible disclosure policy for external researchers

## Reporting Process

1. Identify potential weakness
2. Report via internal ticketing system or security email
3. Do not attempt to verify the weakness (avoid exploitation)
4. Await confirmation and instructions from security team

## Check Details

| Field | Value |
|-------|-------|
| **Owner** | Information Security Officer |
| **Frequency** | Monthly |
| **Instruction** | Review reported weaknesses and ensure proper follow-up |
