If you suspect or witness a security incident, report it immediately. **Quick reporting helps minimize damage.**

## What is a Security Incident?

A security incident is any event that:
- Compromises the confidentiality, integrity, or availability of data
- Violates security policies
- Puts patient data at risk
- Could damage MEDrecord's reputation or operations

### Examples of Incidents

| Category | Examples |
|----------|----------|
| **Data Breach** | Unauthorized access to patient data, lost devices with data |
| **Phishing** | Clicking malicious links, entering credentials on fake sites |
| **Malware** | Virus infection, ransomware, suspicious software |
| **Physical** | Tailgating, lost access badges, stolen equipment |
| **Policy Violation** | Sharing passwords, unauthorized data transfer |
| **System Issues** | Unauthorized access attempts, suspicious account activity |

## How to Report

### Step 1: Immediate Actions

1. **Don't panic** - but act quickly
2. **Don't try to fix it yourself** - you might make it worse
3. **Preserve evidence** - don't delete emails or files
4. **Disconnect if necessary** - if malware is suspected, disconnect from network

### Step 2: Report via Helpdesk

**Primary reporting channel:**

[Report Incident via Helpdesk](https://helpdesk.healthtalk.ai/#/report-issue)

Select "Bug" or "Security Incident" as the issue type.

### Step 3: Provide Information

Include as much detail as possible:

| Information | Why It Matters |
|-------------|----------------|
| What happened | Helps assess severity |
| When it happened | Helps with timeline |
| Who is involved | Identifies affected parties |
| What systems/data | Determines scope |
| Screenshots/evidence | Aids investigation |

## Emergency Contacts

For **urgent security incidents** (data breach, active attack):

| Contact | Method | When to Use |
|---------|--------|-------------|
| Security Officer | security@medrecord.nl | All security incidents |
| IT Emergency | +31 (0)XX XXX XXXX | System compromises |
| Management | management@medrecord.nl | Major breaches |

## After Reporting

1. **You'll receive confirmation** within 1 hour during business hours
2. **The Security Officer will assess** the incident severity
3. **You may be contacted** for additional information
4. **Follow any instructions** provided by the security team

## Severity Levels

| Level | Description | Response Time |
|-------|-------------|---------------|
| **Critical** | Active breach, patient data exposed | Immediate |
| **High** | Potential breach, malware detected | < 1 hour |
| **Medium** | Policy violation, suspicious activity | < 4 hours |
| **Low** | Minor issue, awareness concern | < 24 hours |

## Confidentiality

- All reports are treated confidentially
- You will not be penalized for good-faith reporting
- Anonymous reporting is available if needed
